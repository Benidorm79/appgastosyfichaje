# Revisión de seguridad 2026

Se han añadido cabeceras de seguridad, bloqueo de listado de directorios, protección de archivos de configuración/SQL/documentación, validación CSRF en los nuevos procesos, consultas preparadas, control de permisos por usuario, validación estricta de imágenes, límites de tamaño, claves de API solo en servidor y salida HTML escapada.

Ninguna aplicación conectada a Internet puede considerarse invulnerable. Deben mantenerse HTTPS, copias de seguridad, PHP actualizado, credenciales rotadas, permisos mínimos en Make/Drive y revisiones periódicas.
